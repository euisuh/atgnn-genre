## 1. What ATGNN did (Singh et al., IEEE SPL 2024)

ATGNN treats audio tagging as a graph problem. The key idea is that audio patches
and class labels are jointly modelled as nodes in a graph, rather than treating
classification as a simple linear projection from a fixed embedding.

**Architecture:**
- A CNN backbone extracts a feature map from a log-mel spectrogram → flattened into a set of *patch nodes*
- A **Patch Graph Network (PGN)** builds a dynamic k-NN graph in feature space and runs max-relative graph convolution over patch nodes (captures local spectral/temporal context)
- A **Patch-Label Graph (PLG)** connects patch nodes to label nodes, so each label aggregates relevant patch evidence
- A **Label-Label Graph (LLG)** with a learnable soft adjacency matrix lets labels exchange information with each other
- Final predictions: dot product between refined label embeddings and a learned output matrix

**What it does well:** strong multi-label tagging by exploiting co-occurrence structure between labels. Beats CNN+attention baselines on AudioSet and VGGSound.

**What it does not do:**
- Labels are treated as a flat set — no awareness of semantic hierarchy (genre ≠ sub-genre)
- Label embeddings are randomly initialised — no linguistic prior
- Audio-only — no cross-modal grounding

## 2. What H-ATGNN introduces (this project)

Three orthogonal extensions on top of ATGNN, each switchable independently:

### 2a. Text LM initialisation (`--text_init`)
Label embeddings are seeded from `sentence-transformers/all-mpnet-base-v2` instead
of random. This gives the model a semantic geometry from the start — "bebop" starts
close to "post-bop", "death metal" starts close to "black metal". The embeddings
remain trainable; the LM just provides a better starting point.

### 2b. Hierarchical graph structure (`--hierarchy`)
The flat PLG and LLG are replaced with level-aware variants:

- **H-PLG**: three-level message passing — patches → mood nodes first, then patches + mood → genre nodes, then patches + mood + genre → sub-genre nodes. Each level only receives information from its ancestors, respecting the DAG direction.
- **H-LLG**: the learnable adjacency matrix is masked by a binary hierarchy mask. Within-level edges are unconstrained; cross-level edges only exist along DAG edges (genre ↔ its mood parent, sub-genre ↔ its genre parent). Direct mood → sub-genre shortcuts are forbidden.
- **Consistency loss**: a soft penalty `max(0, p_child − p_parent)` is added to the BCE loss — if a sub-genre fires, its parent genre should also fire.

Ontology used: 8 moods → 15 genres → 48 sub-genres (derived from MTG-Jamendo + MusicBrainz).

### 2c. MERT backbone (`--backbone mert`)
Replaces the lightweight CNN with `m-a-p/MERT-v1-95M`, a music-specific
self-supervised model trained with masked modelling on large-scale music audio.
MERT outputs frame-level representations at 24 kHz which are projected and
adaptive-pooled into 512 patch nodes, then fed into the same PGN/PLG/LLG pipeline.
Last 2 transformer layers are fine-tuned; the rest are frozen.

This gives a second, stronger baseline for the paper (Table 2).

### Core publishable claim
The **rare sub-genre gap** — `mAP_common − mAP_rare` — narrows most dramatically
for H-ATGNN compared to all baselines and AST. Hierarchical context propagates
signal from well-represented parent genres down to rare sub-genres.
