## Bugs encountered and fixed

### 1. `STORAGE_ROOT` wrong default
**Problem:** `.env` and `run_experiment.sh` defaulted to `/runpod-volume`, but RunPod Network Volumes actually mount at `/workspace`.
**Fix:** Updated `.env` and `run_experiment.sh` default to `/workspace`.

---

### 2. Tar files not deleted after extraction → disk full
**Problem:** `os.remove(tar_path)` was inside the `else` block:
```python
if result.returncode != 0:
    print("ERROR")
else:
    print("Extracted")
    os.remove(tar_path)   # ← only ran on success
```
Any non-zero tar exit (even a benign warning) left the `.tar` on disk. With 20 chunks × ~5GB each, disk filled to ~93GB before all chunks extracted.
**Fix:** Moved `os.remove` outside the `else` — always remove after extraction.

---

### 3. Tar extraction ownership error on RunPod
**Problem:** Running as root, `tar -xf` tried to set file ownership to `uid 1000` (original creator) and failed:
```
tar: Cannot change ownership to uid 1000, gid 0: Operation not permitted
```
This caused non-zero exit, which (before fix #2) left tars on disk.
**Fix:** Added `--no-same-owner` flag to tar command.

---

### 4. Download script re-downloading already-extracted chunks
**Problem:** Skip check was `if os.path.exists(tar_path)` — only skipped if the `.tar` file existed. After tars were deleted post-extraction, every restart re-downloaded everything from scratch.
**Fix:** Added check for extracted directory: if `audio/XX/` exists and is non-empty, skip.

---

### 5. Two smoke test runs started simultaneously
**Problem:** First SSH-launched `train.py` was not killed before launching a second one via `nohup`. Both registered W&B runs, both used GPU memory (~18GB each).
**Fix:** Always use `nohup` from the start. Kill existing processes before re-launching.

---

### 6. Corrupt audio file crashed training
**Problem:** A bad `.mp3` file in the dataset caused `soundfile.LibsndfileError` inside a dataloader worker, crashing the entire training run with exit code 1.
**Fix:** Wrapped `_load_audio` in try/except — returns a silent zero waveform for corrupt files instead of crashing.

---

### 7. Chain script used hardcoded PID
**Problem:** Ad-hoc `run_chain.sh` created on the pod used `kill -0 3051` (hardcoded smoke test PID). Fragile across restarts and pod recreation.
**Fix:** Created proper `cloud/run_chain.sh` in the repo — waits by process name (`pgrep -f download_mtg_jamendo`), not PID.
