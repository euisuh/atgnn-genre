## 1. Q&A: Where does the hierarchy (mood → genre → sub-genre DAG) actually happen?

Two separate modules handle it: H-PLG and H-LLG. They work on different parts of the graph.

### H-PLG (Hierarchical Patch-Label Graph) — `models/hatgnn.py:95`

This is where patches communicate *up* to label nodes, level by level.

Instead of every label node aggregating from patches simultaneously (flat PLG), H-PLG does it in three ordered passes:

1. **Mood nodes** pull context only from patches — no label-to-label information yet
2. **Genre nodes** pull from patches *and* the already-updated mood nodes
3. **Sub-genre nodes** pull from patches, mood nodes, and the already-updated genre nodes

Each pass uses a max-relative aggregation: find the k nearest patch (or label) nodes in feature space, compute `max(neighbour - self)` over those neighbours, concatenate with self, then project. The result is that a sub-genre node's representation has been shaped by relevant patches *and* its ancestor mood and genre signal — before any label-to-label graph runs.

The hierarchy is enforced structurally: sub-genre nodes cannot skip genre and pull directly from mood in a shortcut. Information flows strictly downward through levels.

### H-LLG (Hierarchical Label-Label Graph) — `models/hatgnn.py:163`

This runs *after* H-PLG and handles label-to-label message passing, but with a mask.

In the original ATGNN, the LLG adjacency matrix `A` is fully learnable — any label can attend to any other label. H-LLG multiplies `A` by a binary `hierarchy_mask` before use:

```
A_masked = sigmoid(A_raw) * hierarchy_mask
```

The mask allows:
- Within-level edges freely (genre ↔ genre co-occurrence is unconstrained)
- Cross-level edges only along DAG direction (genre ↔ its mood parent; sub-genre ↔ its genre parent)
- Forbidden: mood → sub-genre direct edges (must go through genre)

This means co-occurrence patterns within a level can still be learned (e.g. Jazz and Bossa Nova tend to co-occur), but cross-level attention is constrained to respect the ontology.

### Consistency loss — `models/hatgnn.py:464`

A third enforcement mechanism at training time. After sigmoid, if a sub-genre fires (high probability), its genre parent should also fire. The penalty is:

```
loss += λ * mean(max(0, p_child − p_parent))
```

This is soft — it does not hard-zero child predictions when the parent is absent, but it penalises the disagreement during training. `λ=0.5` by default.

**Summary:** H-PLG enforces hierarchy in how patch evidence reaches label nodes (bottom-up, level by level). H-LLG enforces it in how labels communicate with each other (masked adjacency). The consistency loss enforces it in the output probabilities (training signal).

## 2. MuQ backbone — meeting discussion and plan

### What MuQ is

`OpenMuQ/MuQ-large-msd-iter` is Tencent AI Lab's music self-supervised model. Like MERT it outputs frame-level embeddings from raw audio, but its pretraining objective uses RVQ-discrete tokens (EnCodec/SoundStream-style) as targets rather than MERT's continuous masked modelling targets. Installed via `pip install muq`.

### Plan agreed

1. **Do not swap out MERT pre-emptively.** The paper story (CNN for Table 1, SSL model for Table 2) stays clean.
2. **`SSLBackbone`** generalises over MERT and MuQ — selectable via `--backbone [mert|muq]`.
3. **Run MuQ as a side experiment** (Exp 2) once the main pipeline is working on GPU.
4. Promote based on results:
   - MuQ > MERT → use MuQ as the Table 2 anchor
   - MuQ ≈ MERT → add as an extra row in Table 2
   - MuQ < MERT → drop it

### MuQ-MuLan: replaces CLAP (Exp 3)

Currently MERT (backbone) and CLAP (cross-modal) are from different model families with no shared representation space. `OpenMuQ/MuQ-MuLan-large` maps music and text into the same embedding space — built on top of MuQ. Using MuQ + MuQ-MuLan means both components share a consistent representation space, and CLAP is dropped entirely.

### Key metric to watch

Beyond mAP — the **rare sub-genre gap**:

```
gap = mAP_subgenre_common − mAP_subgenre_rare
```

This should be smallest for `full_hatgnn` / `hatgnn_muq_muqmulan` and largest for `baseline`. This is the core paper claim: hierarchical context propagates signal from well-represented parent genres down to rare sub-genres.
